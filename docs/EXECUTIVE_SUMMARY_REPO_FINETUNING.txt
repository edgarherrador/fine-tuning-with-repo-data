================================================================================
4-WEEK IMPLEMENTATION PLAN
================================================================================

WEEK 1: Data Extraction & Analysis
├─ Extract commits (5K-10K examples with diffs)
├─ Extract PRs + reviews from GitHub
├─ Correlate commits → PR outcomes → reviewer feedback
├─ Identify curriculum progression (simple → complex)
└─ Create training/validation/test splits

WEEK 2: Supervised Fine-Tuning (ExpRL Mid-Training)
├─ Mid-train on MERGED commits only
├─ Curriculum: stage commits by complexity
├─ Input: commit message + context
├─ Output: the actual diff that was applied
├─ Result: Model learns "how good code is written here"

WEEK 3: Feedback Learning (ContextRL) + RL (HAW)
├─ Teach model to parse PR review comments
├─ Learn sparse rewards from PR outcomes (merged/rejected)
├─ Hierarchical Advantage Weighting identifies key tokens
├─ 3-5x more efficient than naive RL
└─ Result: Model learns what reviewers care about

WEEK 4: Optional Value Axis + Deployment
├─ Extract internal confidence signals
├─ Deploy with "reject low-confidence outputs"
└─ Ready for pilot testing
